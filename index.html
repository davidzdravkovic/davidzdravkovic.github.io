About

I am a software developer based in Skopje, Macedonia. I build real-time networked applications and the algorithms underneath them — without frameworks doing the heavy lifting.

When I design something I start from what must always be true — the invariant — then build the smallest piece that guarantees it, then compose outward. The same discipline that produced correct message ordering on the server produced correct view merging and optimistic reconciliation on the client. See the writing section for worked examples.

My main language is C++. I also write JavaScript and React for the client side, and I have shipped a production Android app using Capacitor.

Projects

ChatMe

End-to-end messaging — custom C++ WebSocket server, React client, PostgreSQL, live in production

Built a production chat system from scratch without an application framework. The hard part is the backend: async TCP/WebSocket in C++, ordered delivery under concurrency, JWT sessions, PostgreSQL, and scheduling that keeps the socket path responsive while database work runs on worker threads.

Key engineering work

  • Custom async stack (Boost.Asio, Boost.Beast) — sessions, framing, lifecycle; no framework server
  • Ordered delivery and concurrency control across parallel requests and background work
  • WebSocket hot path isolated from PostgreSQL-backed handlers
  • JWT-authenticated WebSocket upgrades and persistent chat data
  • React optimistic UI with server ACK reconciliation
  • Presence, typing, seen receipts, media uploads; Linux (nginx, systemd) and Android (Capacitor)

Stack: C++, Boost.Asio, Boost.Beast, PostgreSQL, pqxx, React, Vite, JWT, Capacitor

Hardliq

Productivity web app with live progress tracking — deployed and in use

You keep the freedom to shape how you work. I build the structure that makes it hold. Hardliq is for people who want a clear picture of their workload: what is still open, what they finished, and what they chose to drop — updated as they work, not at the end of the week.

Highlights
