About

I am a software developer based in Skopje, Macedonia. I build real-time networked applications and the algorithms underneath them — without frameworks doing the heavy lifting.

When I design something I start from what must always be true — the invariant — then build the smallest piece that guarantees it, then compose outward. The same discipline that produced correct message ordering on the server produced correct view merging and optimistic reconciliation on the client. See the writing section for worked examples.

My main language is C++. I also write JavaScript and React for the client side, and I have shipped a production Android app using Capacitor.

Projects

Real-Time Chat Platform

Messaging system I built end to end — custom C++ server, React client, PostgreSQL, live deploy

Production-ready chat I built from scratch — backend first. The unusual work is the custom asynchronous TCP/WebSocket stack in C++: ordering under concurrency, JWT-backed sessions, PostgreSQL persistence, and scheduling so the WebSocket path stays responsive while database-heavy handlers run on worker threads. React client with optimistic sends and ACK reconciliation; shipped on Linux (nginx, systemd) and Android (Capacitor).

Key engineering work

  • Custom async networking stack (Boost.Asio, Boost.Beast) — framing, sessions, lifecycle; no framework server
  • Ordered message delivery and concurrency control across overlapping requests and background work
  • Dispatch and scheduling that isolate PostgreSQL-backed handlers from the WebSocket hot path
  • Cursor-based pagination over large histories (getBefore / getAfter)
  • JWT authentication, session binding, and authenticated WebSocket upgrades
  • Optimistic UI in React with server acknowledgement reconciliation
  • Presence, typing indicators, seen receipts, and media upload lifecycle
  • Production Linux deploy (systemd, nginx); Android packaging via Capacitor

Stack: C++, Boost.Asio, Boost.Beast, PostgreSQL, pqxx, React, Vite, JWT, Capacitor
